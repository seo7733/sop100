프로젝트 개요
시스템 목적, 구성 파일, 기술 스택
프로젝트 목적 소방청 재난현장 표준작전절차(SOP 100~113)를 웹기반 교육교재로 제작하여 전국 소방관 및 시민에게 무료 공개. AI 챗봇을 통한 질의응답, 커뮤니티 게시판을 통한 현장 경험 공유.

반복 수정 및 심화

무전 시나리오, 화학사고 실습, 자격인증제, 용어해설 등을 추가하며 대화를 통해 점진적으로 완성. Chapter 번호 체계(Ch.1~Ch.6) 정리.

효과적인 프롬프트 팁 "기존 스타일(색상, 폰트, 레이아웃)을 유지하면서 ○○ 내용을 추가해줘" 방식으로 누적 수정. 파일을 첨부하면 Claude가 기존 코드를 읽고 정확히 수정해줌.

Supabase DB · Function 설정
데이터베이스 테이블, RLS 정책, Edge Function 배포

① 테이블 생성 SQL

-- 접속 로그 CREATE TABLE visit_logs ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, page text, ip text, email text, user_name text, avatar_url text DEFAULT '', visited_at timestamptz DEFAULT now() ); -- 게시판 -- (Supabase 대시보드 Table Editor에서 생성 또는 자동 생성) -- posts, comments, blocked_users 테이블 -- 챗봇 로그 CREATE TABLE chat_logs ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, email text NOT NULL, user_name text, question text, answer text, tokens_used integer DEFAULT 0, created_at timestamptz DEFAULT now() );

② RLS 정책 SQL

-- visit_logs ALTER TABLE visit_logs ENABLE ROW LEVEL SECURITY; CREATE POLICY "insert_only" ON visit_logs FOR INSERT WITH CHECK (true); -- chat_logs ALTER TABLE chat_logs ENABLE ROW LEVEL SECURITY; CREATE POLICY "insert_only" ON chat_logs FOR INSERT WITH CHECK (true); CREATE POLICY "본인_사용량_조회" ON chat_logs FOR SELECT TO authenticated USING (email = auth.jwt() ->> 'email'); CREATE POLICY "관리자_챗봇로그_조회" ON chat_logs FOR SELECT TO authenticated USING (auth.jwt() ->> 'email' = 'jkseo1974@gmail.com');

ToDo · 향후 개발 과제
미완성 항목 및 업그레이드 계획

챗봇 학습 강화 (RAG 구축) 우선순위 높음

현재 챗봇은 Claude의 기본 지식 + 시스템 프롬프트로만 동작. index.html 교안 내용을 Supabase pgvector에 벡터로 저장하고, 질문과 관련된 SOP 내용을 검색해서 Claude에 함께 전달하는 RAG(검색 증강 생성) 방식으로 정확도 향상 필요.

챗봇 사용자 계정 변경 방법

현재 하루 20회 제한은 Google 로그인 이메일 기준. 사용자가 다른 Google 계정으로 로그인하면 별도 카운트. 제한 횟수 조정 필요 시 claude-chat Edge Function 코드의 DAILY_LIMIT = 20 값 변경 후 재배포.

Chapter 4 추가 시나리오 보완

현재 Chapter 3 지휘체계 실습에 공사장 화재·화학사고 시나리오 수록. 복합재난, 지하공간, 고층건물 등 추가 시나리오 개발 예정.

학습 이력 관리 시스템

현재 진도는 브라우저 세션에서만 유지. Supabase에 수강 이력(슬라이드별 완료 여부, 퀴즈 점수)을 저장하여 로그인 간 진도 유지 및 관리자 수료 현황 통계 기능 추가.

모바일 최적화

현재 반응형 기본 대응. 소방관 현장 접속 시나리오를 고려한 모바일 특화 UI 개선(터치 제스처, 오프라인 캐싱 등) 필요.

공무원제안 · 적극행정 공모 출품

전국 확산을 위한 공식 제안 채널 활용. 국민신문고 공무원제안 제출 및 소방청 적극행정 공모 출품. 접속자 수·수료율 데이터를 정량적 근거로 활용.
